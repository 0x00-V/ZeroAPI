ZeroAPI Docs

Root endpoint: GET / — Redirects users to documentation

Documentation endpoint: GET /docs

Users

GET /users — List all registered users

POST /users/register — Register a new user curl -X POST /users/register \ -H "Content-Type: application/json" \ -d "{\"Name\":\"test\",\"Password\":\"test\"}"

POST /users/login — Authenticate and receive an access token and refresh token curl -X POST /users/login \ -H "Content-Type: application/json" \ -d "{\"Name\":\"test\",\"Password\":\"test\"}"

POST /users/refresh — Exchange a refresh token for a new access token curl -X POST /users/refresh \ -H "Content-Type: application/json" \ -d "{\"RefreshToken\":\"YOUR_REFRESH_TOKEN\"}"

POST /users/logout — Revoke a refresh token (log out) curl -X POST /users/logout \ -H "Content-Type: application/json" \ -d "{\"RefreshToken\":\"YOUR_REFRESH_TOKEN\"}"

Protected

GET /protected — Test protected endpoint (requires JWT) curl /protected \ -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Notes:

  • All protected endpoints require a valid JWT access token
  • Invalid or expired tokens return 401 Unauthorized
  • Access tokens are short-lived; use refresh tokens to obtain new ones
